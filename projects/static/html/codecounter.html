A Python script I wrote to count various lines in my scripts. This script counts up all the code, comment, & blank lines. It also counts inline-comments, classes, functions, ifs, trys, and some more things. It has the ability to list all imports, variables, caught exceptions, and docstrings. This was one of my first projects, and I may have went overboard a little, but some of this is valuable information to have. I still use this script when I'm making bigger projects, to keep track of my percentage of comments to code (which CodeCounter displays. It will display all totals by default, but can be told to display only the basic totals (Total Lines, Code, Comment, Blank, DocString) using the -t flag. Anyway, here's codecount.py.

DocStrings

With -s passed as an argument codecount.py will output all docstring lines: 

Finished analyzing: codecount.py...
  10 docstrings found in codecount.py:
        * Counts:
        code lines, comment lines, blank lines, inline comments, doc strings,
        doc string lines, imports, classes, functions, try statements,
        exception catchers, print statements, if statements, variables,
        variable assignments
        * Reports count/percentage of each item on the console

        * With use of argument flags i, v, e, and d:
        *     Lists all imports, variables, and exception catchers,
        *     Lists can be sorted in the order of discovery, or alphabetical
        Analyze file, find all tags

* codecount.py will not list duplicate entries, except for docstrings. This is by design. If duplicates are found it will print "1 of 2 unique exceptions found", hence the word "unique"...

**When using -vies, you may also use -d to report items in the order they are discovered, the default is sorted.

Usage

This script was designed using Linux (Linux Mint), but will also work for Windows. Obviously you must have Python installed, if you do not have python installed you may find instructions on how to download it here. To run the script locate your python executable (python.exe, or pythonw.exe in Windows, /usr/bin/python in Linux) and run it like this (Linux method shown): 

python /directory/codecount.py [-hitvesd] script_file_to_analyze.py
OR:
python /directory/codecount.py [-hitvesd] /directory/with/pyfiles

That's about it, I hope this script helps you in some way. For anyone wanting to learn about string parsing or basic python syntax like the if statement, list objects, or file input, this might be a good script to look at and learn from.
