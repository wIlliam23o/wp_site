{# Description #}
Generates skeleton code from a glade file. The generated signal handler stubs can then be filled in to provide functionality. Arguments for signal handlers are automatically decided through introspection using Gtk.
I didn't know this before I wrote it, but this does the exact same thing that [bash]pyqt4uic[?bash] and [bash]pykde4uic[?bash] do for Qt designer files. It does a little more actually. It includes a GUI for previewing/editing, and also generates stubs for the signal handlers. It does this by reading the glade file's XML, parsing it, and using introspection to generate the stubs.
The glade file still needs to be present in the finished application directory. It is used with [py]Gtk.Builder.add_from_file()[/py] to generate the widgets.
When everything is working as expected, doing [bash]glader input.glade output.py[?bash] will create an executable Gtk application that can be ran and previewed by running [bash]./output.py[?bash]

{# Command Line #}

Command Line

If an input file name (glade), and an output file name (python) are passed to [bash]glader[?bash], the code will be generated and written to the output file.
You can pass [bash]-[?bash] as the output file name to write to stdout.
{# Command Line Help #}

Command Line Help

    Usage:
        glader.py -h | -v
        glader.py [FILE] [OUTFILE] [-d] [-g]

    Options:
        FILE           : Glade file to parse.
        OUTFILE        : File name for output.
                         If - is given, output will be printed to stdout.
        -d,--dynamic   : Use dynamic object initialization method.
        -g,--gui       : Force use of a GUI, even when an output file is given.
                         You still have to use the 'Save' button to apply
                         changes.
        -h,--help      : Show this help message.
        -v,--version   : Show version.
{# GUI Mode #}

GUI Mode

When [bash]glader[?bash] is ran with no arguments, or just an input file is given, a GUI is loaded. In GUI mode a preview is generated, and can be edited before saving. When an input file is given, or [bash]--gui[?bash] is used, the preview code is automatically generated when the program loads.
The GUI supports Python syntax highlighting using GtkSourceView. The viewer uses GtkSourceView themes, and can be changed using the theme selector. Themes are located in [bash]/usr/share/gtksourceview-3.0/styles[?bash], and can be downloaded from various places on the internet. The most common is https://wiki.gnome.org/Projects/GtkSourceView/StyleScheme .
