Linux command-line tool that opens files with your favorite editor, but also automatically uses your favorite sudo command when needed for write-permissions. This way, you won't forget to open with sudo and get that annoying "no-permissions" or "cannot save" message.

If root-permissions are required for write-access, your elevation command will ask you for the password. If they are not required, your editor command will be used alone. This is because opening all files as root may put root ownership/group status on files that don't really need it, which is also annoying.

Usage

{% acesnippet "cedit-usage-example" ".bash" %} cedit # ...to open a file. cedit --list # ...lists current configuration (editor and elevation command) cedit --editor path_to_editor cedit --elevcmd path_to_elevcmd # ...sets your favorite commands so cedit will remember what to use. {% endacesnippet %}

If your favorite editor isn't in the same directory as cedit or /usr/bin, then you will need to set the editor like this: {% acesnippet "cedit-abspath-example" ".bash" %} cedit -e /my/path/to/myeditor {% endacesnippet %}

Installation

You can create a symlink to the cedit.py script yourself, or you can use the builtin 'install' command.

To install cedit in /usr/bin (for all users) run:

{% acesnippet "cedit-install-example" ".bash" %} sudo ./cedit.py -i {% endacesnippet %}

To install cedit for a single user run:

{% acesnippet "cedit-install-user-example" ".bash" %} ./cedit.py --install --user # this will try to install cedit to /home/USERNAME/.local/bin # you will need this directory to exist, and be included in your PATH variable. {% endacesnippet %}

If everything went well, or you manually created the symlink yourself, you should be able to run cedit like this from anywhere:

{% acesnippet "cedit-run-example" ".bash" %} cedit # with no .py extension or ./ prefix... {% endacesnippet %}

With no settings

If no settings are set, cedit will try to look for a few 'popular' commands to work with.

cedit will look for these editors if your favorite isn't set:

  • kate
  • gedit
  • leafpad
  • kwrite

cedit will look for these elevation commands if your favorite isn't set:

  • kdesudo
  • gksudo
  • sudo

Without any settings, if you have both kate and kdesudo installed, cedit will open your 'root-permissions' file by running:
{% acesnippet "root-perm-example" ".txt" %} kdesudo kate filename {% endacesnippet %}
If the file doesn't require root permissions then just:
{% acesnippet "normal-perm-example" ".txt" %} kate filename {% endacesnippet %}
