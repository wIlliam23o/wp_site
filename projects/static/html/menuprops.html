A freedesktop.org menu editor designed for Linux desktop environments such as Gnome, XFCE, LXDE, Cinnamon, & KDE. Created with Python 2.7, GTK 2, & Glade 3. Menu Props allows you to view, create, delete, and modify launcher files (.desktop), directory files (.directory), and menu files (.menu).

There are two main sections in Menu Props, the first is called the 'Menu' section. This is where you can view the 'menu tree' and modify it's structure and/or contents. This section is very similar to 'alacarte-style' menu editing, except has more features and options. Menu Props has a built-in properties editor for launchers and directories, and does not require an external command such as 'desktop-file-edit' to make basic changes. You can still choose to open items in an external editor using the context-menu (right-click) if you like. The default external editor can be changed in "Settings".

The 'Items' section lists all launcher/directory files that are found in the default XDG directories, and any other directory found in the XDG_DATA_DIRS environment variable. This allows you to make changes to items that may not be loaded in the menu tree. The 'Items' section is split into 'Entries', 'Dirs', and 'Search'. Here you can find all launchers or directories listed in order by name, and search both lists.

The 'Search' section lets you search all launchers/directories for text inside the file contents. All results can be viewed or modified the same way, as if you were looking at them from the 'Items' or 'Dirs' list.

Features

  • Display menu tree
  • Move, Edit, Hide, Show, Delete, Create Menus
  • Move, Edit, Hide, Show, Delete, Create Launcher/Directories
  • Display launcher/directory information
  • Delete launcher/directory-file currently being used
  • Delete launcher/directory-file permanently from all XDG directories
  • Automatic Back-Up of Menu File on first run
  • Run As.. (Normal User or Root)
  • Change launcher execution (Always launch as Normal User/Root)
  • Export launcher/directory list to text file (Full Mode, all info found in Description)
  • Export launcher/directory list to text file (Simple Mode, Names & Filenames)
  • Edit launcher/directory using external editor (User has choice of external editor in Settings)
  • Find all locations of launcher/directory file (Scans all XDG directories)
  • Validate launcher/directory file using desktop-file-validate (If "desktop-file-validate" is installed)
  • Search launcher/directory content
  • Optional backup of edited/deleted files
  • Default menu file can be set in settings (If Menu Props fails to detect the correct menu file)
  • Default elevation command can be set in settings (For kdesu, or other command users)
  • Edit local/system menu file using external editor
  • Open launcher/directory files from another directory for viewing/editing
  • Save menu as another filename
  • Check for update
  • Refresh list (Manually refresh the menu/items)

Installation:

Installation notes are in a file called 'installation-notes.txt' inside the package. Basically, I made an install & uninstall shell script that copies the necessary files to the appropriate location. To run it, open a terminal, 'cd' to the /menupropsX.X.X directory, and type:
[code]sudo ./install[?code]
If you do not wish to install the application you can always try it first by opening a terminal, 'cd' to the /menupropsX.X.X directory, and type:
[code]./menuprops[?code]
The install script just ensures that a Menu Props icon gets placed in the applications menu, and that the icon, desktop file, launch script, etc. are placed in their correct directory. If 'install', 'uninstall', or 'menuprops' fails to run be sure that they are set to executable using [code]sudo chmod a+x [script name][?code]. The 'helper script', Menu Props Writer, needs root privileges due to the directories Menu Props must access. You will be need to know root's password.

Known Bugs

There are still a few bugs in this version that I am currently working to fix. I'll list them here as a warning. I have tested Menu Props with several different desktop environments, under several different distros of Linux. If any bugs occur, or you wish to port Menu Props to your favorite environment, please send me a line at cj@welbornprod.com with your current Linux distribution & Desktop Environment name/version. I will try my best to correct the issue if possible.

  • KDE does not move items up or down. KDE uses an alphabetical listing as the default, so trying to move items up or down may not affect the Main Menu. (Even if it shows up in Menu Props.)
  • Cinnamon does not display sub-menus. This is Cinnamon's design, not mine. Any sub-menus created will show their items in either the "Other" menu, or the parent menu. I discourage the use of sub-menus in the Cinnamon environment.
  • Setting OnlyShowIn causes items to disappear from Menu Props on certain systems, with certain values set. (Setting OnlyShowIn=KDE; using KDE will cause this.) I believe this is a limitation of gmenu (gnome-menus), which Menu Props is built with. I am currently working to remove this dependency (see Future). To fix items that have disappeared: Open the launcher from the "Items" tab, and remove the OnlyShowIn option.
  • Some systems don't have gmenu - Some systems need a little help to get gmenu going, here is some good info on how to get it for your system. ( instructions are for Fedora, but it works for Slackware. It may work on other systems too.): gmenu editing fix

** If Menu Props shows duplicates this may not be a bug. Menu Props will always show duplicates for certain applications, this is by design. For example, Konqueror shows duplicate entries in Menu Props, but not in the Main Menu. This is by Konqueror's design, not mine. They register several launchers with different file/url types. Several other applications do this also. It's not an actual duplicate unless the launcher .desktop files show the exact same name. Always check the launcher's .desktop file name, if they're slightly different then it's not a bug.

Future

Except for possible bug-fix releases, Menu Props is frozen until the completion of the next project ("unimenu", working title). This project aims to do at least two things. The first is to rid Menu Props of its gmenu-dependency, and the second is to provide an open-source freedesktop.org compliant module for python. At the start of the Menu Props project I quickly realized that there were few options for editing menus in python, especially if you are targeting multiple desktop environments. Gmenu was made for Gnome, and is not guaranteed to work for other environments. It's actually not guaranteed to work at all (specifically states that API's may change, and cannot be held accountable for changes that break applications). Now that I have a better understanding of how freedesktop.org works, I believe I can write a reliable module to be used in Menu Props or any other menu-related project for python. I will update Menu Props and this site as the project progresses.
